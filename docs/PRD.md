# 讀書筆記本（小說專用） 產品需求文件 (PRD)

## 1. 專案概述

- **背景與動機**：喜愛看小說的讀者通常擁有龐大的閱讀量，但時間一久，常會忘記精彩的劇情發展、角色魅力或是當時看完的感動。本系統專門為小說愛好者打造，提供一個簡單直覺的平台，讓讀者能隨手記錄小說的書名、作者、讀後感以及個人評分。
- **目標用戶**：熱愛閱讀各類小說（如：文學小說、輕小說、網路小說等），希望記錄每部小說評價與心得的讀者。
- **核心價值主張**：提供一個輕量化、易搜尋的專屬小說紀錄庫，幫助讀者保存對故事的感動與評價，不再忘記讀過的好書。

## 2. 功能需求

以下為系統的主要功能清單，並透過「使用者故事」來描述需求：

1. **新增小說筆記**
   - *作為一名小說讀者，我希望能夠新增一筆小說的閱讀紀錄（包含書名、作者、讀後心得、1-5 星評分），以便將剛看完的故事做個總結並保存下來。*
2. **瀏覽筆記列表**
   - *作為一名小說讀者，我希望在首頁可以看到我所有記錄過的小說清單，以便快速回顧我最近看了哪些小說以及給予的評價。*
3. **搜尋小說筆記**
   - *作為一名小說讀者，我希望首頁能有一個搜尋欄，可以透過輸入「書名」或「作者」進行搜尋，以便在茫茫書海中快速找出我想回味的那本小說紀錄。*
4. **編輯既有筆記**
   - *作為一名小說讀者，我希望點擊某篇筆記後能夠修改讀後心得或更新評分，以便在重新二刷、三刷這部小說產生新體會時能夠隨時補充。*
5. **刪除筆記紀錄**
   - *作為一名小說讀者，我希望能夠刪除特定的小說筆記，以便在輸入錯誤或不想保留該筆紀錄時，保持系統的整潔。*

## 3. 非功能需求

- **技術限制**：
  - 前端頁面：HTML / CSS 搭配 **Jinja2** 模板引擎進行渲染。
  - 後端伺服器：使用 Python + **Flask** 框架開發。
  - 資料庫：使用 **SQLite** 輕量級關聯式資料庫儲存筆記紀錄。
- **效能考量**：
  - 確保在單機測試環境下，頁面切換與搜尋結果的呈現應在 1 秒內完成。
- **安全考量**：
  - 表單輸入（特別是評分功能）需進行基礎的前後端資料驗證（確保評分為 1-5 之間）。
  - 對資料庫的查詢應避免 SQL Injection 風險（可透過 ORM 或參數化查詢進行防護）。

## 4. MVP (最小可行性產品) 範圍

為了確保專案能如期完成，我們將功能區分為不同優先級別：

- **Must Have (必備)**
  - 首頁小說筆記清單展示。
  - 新增小說筆記（欄位：書名、作者、讀後心得、1-5 星評分）。
  - 關鍵字搜尋功能（書名或作者）。
- **Should Have (應該要有)**
  - 編輯現有的小說筆記內容。
  - 刪除現有的小說讀書筆記。
- **Nice to Have (加分項目，時間充裕再做)**
  - 加入「小說類型」標籤（例如：奇幻、推理、愛情等）。
  - 將評分以視覺化的圖示（如星號 ★★★☆☆）來顯示。
