# 讀書筆記本（小說專用） 路由與頁面設計

本文件根據產品需求與系統架構，規劃所有的 API 路由（Routes）以及對應的 Jinja2 前端模板。

## 1. 路由總覽表格

| 功能名稱 | HTTP 方法 | URL 路徑 | 對應模板 | 說明 |
| :--- | :--- | :--- | :--- | :--- |
| 首頁與筆記列表 | GET | `/` | `templates/index.html` | 顯示所有筆記，或處理帶有 `?search=` 參數的搜尋請求 |
| 新增筆記頁面 | GET | `/add` | `templates/form.html` | 顯示供使用者輸入書名、作者等資訊的空白表單 |
| 新增筆記處理 | POST | `/add` | — | 接收表單資料寫入資料庫，成功後重導向至首頁 `/` |
| 編輯筆記頁面 | GET | `/edit/<id>` | `templates/form.html` | 根據 `<id>` 查詢資料，並將現有內容預填至表單中 |
| 編輯筆記處理 | POST | `/edit/<id>` | — | 接收修改後的表單資料，更新資料庫，成功後重導向至首頁 `/` |
| 刪除筆記處理 | POST | `/delete/<id>`| — | 根據 `<id>` 刪除指定紀錄，完成後重導向至首頁 `/` |

## 2. 每個路由的詳細說明

### 首頁與搜尋
- **URL**: `/`
- **方法**: `GET`
- **輸入**: 
  - URL Query Parameter: `search` (可選)
- **處理邏輯**: 
  - 若有 `search` 參數，呼叫 `note_model.search_notes(keyword)` 進行模糊查詢。
  - 若無參數，呼叫 `note_model.get_all_notes()` 取得所有紀錄。
- **輸出**: 渲染 `index.html`，並將取得的筆記清單傳入模板中。
- **錯誤處理**: 若無任何符合的資料，前端頁面顯示「找不到相關筆記」的提示。

### 新增筆記
- **URL**: `/add`
- **方法**: 
  - `GET`: 渲染並回傳 `form.html` (空白表單)。
  - `POST`: 接收表單提交的欄位 (`title`, `author`, `review`, `rating`)。
- **處理邏輯**: 
  - POST 請求發生時，檢查必填欄位是否都有值，且評分為 1~5 之間。
  - 若驗證通過，呼叫 `note_model.create_note(...)`。
- **輸出**:
  - GET: 顯示表單。
  - POST: 成功後 HTTP 302 重導向至 `/`。
- **錯誤處理**: 若 POST 驗證失敗，帶上錯誤訊息重新渲染 `form.html` 並保留原本使用者填寫的內容。

### 編輯筆記
- **URL**: `/edit/<id>`
- **方法**: 
  - `GET`: 根據 `<id>` 查詢該筆資料，並渲染 `form.html` (帶入現有資料)。
  - `POST`: 接收表單提交的更新資料。
- **處理邏輯**:
  - GET 時，呼叫 `note_model.get_note_by_id(id)`。若找不到該筆記，可回傳 404 或重導向首頁。
  - POST 時，驗證資料格式，若成功則呼叫 `note_model.update_note(...)` 進行更新。
- **輸出**:
  - GET: 顯示帶有預設值的表單。
  - POST: 成功後 HTTP 302 重導向至 `/`。

### 刪除筆記
- **URL**: `/delete/<id>`
- **方法**: `POST`
- **處理邏輯**: 
  - 呼叫 `note_model.delete_note(id)` 從資料庫刪除紀錄。
  - 註：使用 POST 而非 GET 是為了防止爬蟲或惡意連結輕易觸發刪除操作。
- **輸出**: HTTP 302 重導向至 `/`。

## 3. Jinja2 模板清單

所有的視圖頁面將放在 `app/templates/` 目錄中：

- `base.html`: 
  - 基礎共用模板，包含網頁的標頭 (`<head>`)、引入的 CSS、頂部導覽列與搜尋欄。所有其他頁面都會繼承此模板。
- `index.html`: 
  - 首頁，繼承自 `base.html`。
  - 負責用迴圈 (`{% for note in notes %}`) 渲染筆記列表卡片，並顯示搜尋結果。
- `form.html`: 
  - 表單頁面，繼承自 `base.html`。
  - 負責新增與編輯的操作。可以根據傳入的 `note` 變數存在與否，決定是要顯示「新增筆記」還是「編輯筆記」，以及決定表單的 `action` 網址。
